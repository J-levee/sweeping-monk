# 扫地僧的内功 —— 科研方法论与思维模型库

> 本文件是扫地僧"内化的海量知识"。老衲于藏经阁遍览群书，对各研究方向的方法论皆通，开口点破问题前，先在此取用相应心法。
> 用法：遇题先回"思维模型库"选切入角度，再回对应章节取具体判读清单。不求全看，只取最贴的一两层。
> 扫地僧强在"内功"（方法论），招式（工具/技术）是末节——先以内功点破，再言招式。

---

## 一、思维模型库（先选角度，再下场）

遇到任何科研难题，先问"该用哪把刀"：

- **第一性原理**：把问题拆到不可再分的基本事实，从那里推，别被行业惯例带着走。适用：方法选型、反驳"大家都这么做"。
- **奥卡姆剃刀**：解释力相当时，假设最少的那个更可能为真。警惕为凑显著性硬加的中间变量。
- **可逆性测试**：这个决定可逆吗？可逆就小步快跑先试；不可逆（如大规模预注册、毁样实验）就先想清再动。
- **类比迁移**：此领域的成熟解法，能否映射过去？注意类比只是启发，边界不同即失效。
- **二阶效应**：A 直接导致 B，但 B 还会引出 C。看结论时多想一步"然后呢"。
- **边界条件思维**：任何结论都有适用前提。先找"它在什么情况下不成立"，比找"它对"更防骗。
- **反证法**：假设结论错，最容易在哪里露出马脚？那往往就是该重点验证之处。
- **分而治之**：大问题拆成可独立验证的小假设，逐个击破，避免"一锅煮"导致归因不清。

## 二、科研方法论内核（通则）

- **演绎 vs 归纳**：演绎从公理推特例（确定性高，但前提要对）；归纳从样本推规律（有概率性，警惕以偏概全）。多数实证研究是"归纳出假设→演绎设计验证"。
- **假设驱动设计**：先有可证伪的假设，再设计能"证伪它"的实验，而非先收集数据再找故事。
- **可证伪性**：一个不能被证据推翻的假设，不是科学假设，只是信念。审稿人最爱问"什么结果会让你推翻结论"。
- **变量与混淆**：明确自变量/因变量/控制变量；混淆变量（同时影响因和果的第三者）是因果推断的头号敌人。
- **内部效度**：结论在你这个样本/设置里成不成立（控混淆、随机、盲法）。**外部效度**：能否推广到别的人群/场景。两者常此消彼长，别混为一谈。
- **预注册**：高风险不可逆的研究，先把假设与分析计划公开登记，防 p-hacking 与事后改写故事。

## 三、研究范式与定位（先定"世界观"）

- **定量 / 定性 / 混合**：定量回答"多少、多大"（统计量）；定性回答"为什么、怎么回事"（意义与机制）；混合用于兼顾广度与深度。选错范式，后面全歪。
- **实证 / 诠释 / 批判范式**：
  - 实证（positivism）：世界客观可测，求因果与规律。
  - 诠释（interpretivism）：现实是社会建构，求理解与意义（访谈、民族志）。
  - 批判（critical）：求权力结构与变革（行动研究、批判话语分析）。
- **判读清单**：作者的范式自洽吗？方法是否匹配其宣称的范式？常有人用定性数据硬做定量结论，破绽就在此。

---

## 四、研究设计类型（选对"阵法"）

- **实验 / RCT（随机对照）**：金标准因果识别。随机分组消混淆；双盲防偏。适用可干预的场景。
- **准实验**：不能随机（伦理/现实限制）时用，靠匹配、差分、工具变量近似控混淆。
- **观察性**：队列研究（前瞻）、病例对照（回顾）、横断面（快照）。只能证相关，因果要谨慎。
- **案例研究**：深挖单/少数个案，适合"如何/为什么"类问题；外推性弱，重典型性与理论饱和。
- **质性方法**：半结构访谈、焦点小组、民族志、扎根理论。重饱和与可信度（信度/效度替代指标：可确证性、转移性）。
- **调查研究**：问卷/量表，重抽样代表性与测量信效度；自陈量表有共同方法偏差风险。
- **行动研究**：实践中迭代改进，重"改"而非"证"，适合教育/组织场景。

---

## 五、研究问题建构（先把"靶"立正）

- **PICO 框架**（临床/干预类）：Population / Intervention / Comparison / Outcome。问题模糊多半是这四样没钉齐。
- **研究问题句式**：避免"关于 X 的研究"这种无边界问法；改成"在 A 条件下，B 对 C 的影响机制是？"
- **概念框架 / 理论模型**：变量间关系先画成图（谁影响谁、中介/调节在哪），再设假设。没有框架的研究像无图下棋。
- **操作化**：抽象概念→可测指标（如"学习投入"→ 每周时长+作业完成率+自评问卷）。操作化不当，构念效度先垮。
- **范围收敛**：问题太大则拆成 2~3 个子问题，逐个击破（分而治之）。

## 六、系统综述与元分析（综览全局之法）

- **PRISMA 流程**：检索→去重→初筛(题摘)→全文筛→纳入→质量评估→合成。报告时画漏斗图与流程图。
- **检索策略**：多库+多词+滚雪球（引文回溯/前向）；留检索式供复现。只搜一个库是破绽。
- **偏倚风险评估**：RCT 用 RoB 2，观察性用 ROBINS-I / Newcastle-Ottawa。忽略偏倚评估的综述站不住。
- **异质性**：I² 高说明研究间差异大，别硬合并；先做亚组/元回归找来源，或改叙述性合成。
- **效应量合并**：用标准化效应量（Cohen's d / OR / RR），报告置信区间，不只报 p。

---

## 七、理论构建（从地到天）

- **演绎 vs 归纳建构**：从理论推假设（演绎），或从数据里长出理论（归纳，如扎根理论）。
- **扎根理论**：开放→主轴→选择性编码，理论饱和即止；忌先有结论再找数据填（那是验证不是建构）。
- **构念与测量**：新构念须证效度（内容/结构/效标）与信度（α/Cronbach、重测）；量表直接翻译要背对背翻译+等值检验。
- **概念模型 vs 因果模型**：模型要标清气路径与未观测混淆，别把相关箭头画成因果箭头。

---

## 八、文献调研方法论

- **broad→narrow 收敛**：先用综述/元分析定位领域地图，再顺引文向前（前向引用=谁引用了它）向后（它引了谁= seminal works）追溯，最后锁定核心几篇。
- **找 seminal works**：看高被引论文的"被引"脉络，反复出现的几篇即是奠基；不要只追新，经典常在。
- **title-abstract triage**：30 秒判读——研究问题是什么？方法大致对路吗？和我的缺口相关吗？不相关立刻放过，省下全文时间。
- **综述文用法**：新手先读 2~3 篇近三年综述建立坐标系；但综述有滞后，关键前沿要看原文。
- **知识库组织**：按"研究问题→方法→结论→局限"四栏记笔记，而非整篇摘抄；用 Zotero/知识库管理，便于写综述时一键调取。

---

## 九、批判性阅读框架（IMRaD 逐段拆）

- **Introduction**：研究 gap 是真 gap 还是作者硬造的？动机是否成立？
- **Methods**：样本量够吗？抽样有偏吗？变量怎么测的（构念效度）？随机/盲法做了没？可复现吗？
- **Results**：报告了所有指标还是只报显著的？效应量多大（别只看 p<0.05）？置信区间宽不宽？
- **Discussion**：结论是数据支撑的，还是过度外推？作者自己说了局限吗？
- **漏洞速查清单**：① 样本量与功效 ② 选择偏倚 ③ p-hacking/多重比较未校正 ④ 相关当因果 ⑤ 效应量 vs p 值 ⑥ 复现性存疑 ⑦ 未报告失败条件 ⑧ 范式/方法错配。

## 十、学术写作结构

- **IMRaD 逻辑链**：每节只回答一个问题，节与节用"因此/然而/由此"粘连，别让评委自己拼图。
- **Contribution 写法**：用"相比现有，本文首次/更好地____"的句式，三条为上限，宁少勿泛。
- **逻辑流自检**：删掉任意一段，前后还连得起来吗？连不起来说明该段是注水或跑题。
- **图表即论证**：每个图表面对一个子论点，正文别重复图表数字，要解释"它意味着什么"。

## 十一、常用统计常识与坑

- **多重比较**：测了 20 个指标，必有几个 p<0.05 纯属偶然。做 Bonferroni / FDR 校正。
- **相关 ≠ 因果**：相关只说明共变；要因果，需随机实验或严谨的因果识别（RCT、IV、断点、DID）。
- **效应量 > p 值**：p 只说"有没有"，效应量说"多大"。p 显著但效应微小，可能无实际意义。
- **功效分析（Power）**：事前算样本量，否则容易得到"假阴性"或"夸张效应"（低功效研究易高估）。
- **离群值**：先查是记录错误还是真信号，别顺手删；做敏感性分析（去不去离群结果稳不稳）。
- **p-hacking**：不断换指标/分组直到显著，会制造假阳性。用预注册和对冲策略规避。

## 十二、审稿意见与开题应对（分类型）

- **方法性质疑**（样本小/设计弱）：补实验或补分析，坦诚局限，说明结论的边界；能补则补，不能补就收窄 claim。
- **创新性不足**：在 Introduction/Related Work 里明确划出"现有没做的、本文新在何处"，必要时补 comparisons 凸显差异。
- **表述问题**（不清楚/图表乱）：这类最易改，逐条照改并标注改处；别辩解，直接改。
- **对立意见**（审稿人之间矛盾）：不讨好一方踩另一方，客观说明两派依据，给出本文取某立场的理由。
- **开题 / 基金逻辑**：立论讲清"研究意义（为何重要）→ 创新点（新在何处）→ 可行性（条件/数据/方法是否够）→ 研究基础（前期积累）"。四样缺一，评委便觉火候未到。
- **总原则**：逐条回复，先谢后答；改了说"已改在第 X 页"，没改说"为何不改"并给依据。态度比文笔更影响结果。

## 十三、学术发表与规范（江湖规矩）

- **审稿流程**：投稿→技术审查→编辑初审→外审（2~3 人）→决定（接收/小修/大修/拒）。读懂决定类型，大修不等于没戏。
- **预印本**：arXiv/bioRxiv 等可抢首发、收反馈；但部分期刊对预印本政策不同，投前查清。
- **署名与贡献**：按实际贡献署名（CRediT 分类），通讯作者担责；挂名/买署名是学术不端。
- **学术诚信**： plagiarism、数据造假、图像不当处理、选择性报告，一律红线。可复现是底线。
- **标识与溯源**：ORCID 统一起作者身份，DOI 固化文献版本；引用务必给到具体出处，别"据研究表明"糊弄。

## 十四、常见科研逻辑谬误

- **幸存者偏差**：只看到"存活"的样本，忽略沉默的大多数（如只研究发表论文，忽略被拒的）。
- **确认偏误**：只收集支持自己假设的证据，无视反例。对策：主动找"能推翻我的"证据。
- **德州神枪手**：先开枪再画靶（数据出来再编假设），把偶然当规律。对策：预注册。
- **因果倒置**：把"相关"理解成 A→B，其实是 B→A（如"喝咖啡的人更焦虑"可能是焦虑的人更爱喝咖啡）。
- **虚假两难**：把连续谱硬说成"要么 A 要么 B"，忽视中间地带与交互效应。
- **诉诸权威/人数**："大牛都这么做""很多人认同"不等于对。回到证据与逻辑本身。
- **过度拟合**：模型在训练集完美、到新数据就垮。警惕为解释噪声硬加复杂度。
- **范式错配**：用质性数据硬做定量结论，或用定量框架苛求质性研究的"代表性"——破绽常在此。

## 十五、贝叶斯方法（现代视角补完）

- **频派 vs 贝叶斯**：频派回答"在重复抽样下，这个结论多稳"；贝叶斯直接回答"给定这些数据，我对某个参数的把握有多大"（后验分布）。两者不矛盾，是同一问题的两种语言。
- **后验 = 先验 × 似然 ÷ 证据**：先有对参数的**先验**信念，用数据（似然）更新成**后验**。先验不是随便拍——弱信息先验（regularizing prior）能防过拟合，强先验必须公开论证来源。
- **何时用贝叶斯**：小样本/组间不平衡、层次模型（多层级收缩）、需要"连续累积证据"（贝叶斯 meta）、或想要"参数落在某区间的直接概率"而非频派 CI。
- **坑**：
  - 先验太强 → 扯着数据走，结论先入为主（警惕"我相信 X 所以后验也 X"）。
  - MCMC 未收敛 → 后验不可信；必看 Rhat（≈1）与有效样本量 ESS。
  - 把 95% 可信区间当成频派置信区间解释——语义不同。
- **与扫地僧**：低功效研究易高估效应（`sources/B4-ioannidis-false-findings.md`），贝叶斯层次模型能把极端估计往总体均值"收缩"，部分缓解，但前提是先验透明、不藏偏。

## 十六、功效分析与样本量规划（实操）

- **事前功效（power）**：给定 α、你预期的效应量、目标功效（通常 0.80），**反推**需要多少样本。不事前算，要么得假阴性（漏真效应），要么因低功效而把效应估得虚高。
- **效应量从哪来**：前人 meta 分析、pilot 研究、或 Cohen 经验值；都没有就做**灵敏度分析**（不同效应量下各需多少 n）。
- **工具**：G*Power、R 的 `pwr`、或直接写模拟（最稳，能纳入你的真实设计）。
- **多重比较下的功效**：做了 FWE/多重校正，所需 n 更大；检验几次要事前规划，别事后补。
- **坑**：
  - 事后"再补样本直到显著" = p-hacking，等价于把 α 偷偷放大。
  - 只说"样本够大"不报事前功效，是空话；评审判的是"你事前算过吗"。
  - 簇随机忘乘设计效应（ICC）→ 功效虚高，样本量严重低估。
- **与 B1/B4**：多重比较（FWE，见 `sources/B1-fwe-neuroimaging.md`）与功效此消彼长——校正越严，越需要更大样本；低功效 + 偏倚 = 多数发现是假的（见 `sources/B4`）。

## 十七、开放科学生态（让研究可复现）

- **预注册（preregistration）**：假设与分析计划**先登记**再收数据（OSF / ClinicalTrials / AsPredicted），防 HARKing（先有结果再编假设）与 p-hacking。**注册报告（Registered Reports）**：方法先送审、接收与否不依结果——从源头切断"结果决定一切"。
- **数据共享（FAIR）**：可发现 / 可访问 / 可互操作 / 可重用；存 Zenodo / OSF / 领域库，给 DOI。光说"数据可向作者索取"不够。
- **代码与材料开源**：分析脚本、实验刺激、问卷、预注册分析计划一并公开，别人才能一键复现。
- **预印本**：arXiv / bioRxiv / OSF Preprints 抢首发、早收反馈；但投前查清目标期刊的预印本政策（少数禁预印本）。
- **可复现自检清单**：原文 + 数据 + 代码 + 环境（包版本）四样齐，别人拉下来能重跑出同样数字，才算可复现。
- **与 B5**：可复现危机不是单人能解的——要方法 + 报告 + 复现 + 评价 + 激励五维协同（见 `sources/B5-manifesto-reproducible.md`）。扫地僧开口点破时，常把"是否预注册/数据是否公开"作为第一道可复现闸门。

## 十八、复现危机与多重比较控制（批判性阅读第一闸门）

- **复现危机本质**：太多"显著"发现无法被独立重复。根因不是某个坏人会造假，而是**系统性方法偏**：低功效 + 偏倚 + 选择性报告（见 `sources/B4-ioannidis-false-findings.md`）。
- **多重比较（multiple comparisons）**：同一个数据集测了 20 个指标，纯偶然就可能有 1 个 p<0.05。必须校正：
  - **FWE（家族-wise 误差）**：Bonferroni（最严）、Holm（逐步，略松）、置换检验（见 `sources/B1-fwe-neuroimaging.md`）。
  - **FDR（错误发现率）**：Benjamini–Hochberg，适合"找候选"的高通量场景，比 FWE 更有力但容更多假阳性。
  - 事先规划检验几次——事后才想起来校正，等于承认在"挑显著的"。
- **p-hacking 家族**：换指标直到显著、删 outlier 直到显著、因变量钓鱼、分阶段补样本直到显著（等价于偷偷调大 α）。对策：预注册（见 `cards/replication-preregistration.md`）+ 报所有检验。
- **批判阅读三连问**（遇"显著"结论先问）：① 做了几次检验、校正了没？② 效应量多大、CI 多宽？③ 数据/代码公开吗、别人复现过吗？三问任一答不上，结论先打折。
- **与扫地僧**：这是老衲开口点破的"第一闸门"——比方法细节更先于一切的是"这发现可靠吗"。

## 十九、质性-量化整合工作流（不止类型学）

- **混合不是"两份报告拼一起"**：整合（integration）发生在**分析层与解释层**，不是附录。常见整合载体：联合展示（joint display / 矩阵）、同一批受访者的量质数据并排、用质结果解释量结果的意外。
- **衔接点设计**：序列式（QUAN→qual 或 qual→QUAN）要在"接口"处留窗口——量的分组如何决定质的取样（极端组/典型组）？质的发现如何回灌量的模型（加调节/中介）？
- **权重与合法化（legitimation）**：写清以谁为主（大写 QUAN/QUAL）；混合专属威胁在"接口"——设计层（两类数据是否可比）、内部层（各自效度）、解释层（合流是否牵强）、外推层（结论靠混合还是单方）。
- **实操清单**：
  1. 先写一句"混合要解决什么单一范式答不了的问题"。
  2. 定主导范式与加权，定接口点与取样规则。
  3. 分别控量（信效度）与质（可信度/可确证性），再加混合效度检查。
  4. 讨论层必须出现"量质合流"段，而非各写各的。
- **与 `cards/mixed-methods.md`**：那张卡给类型学（并行/解释序列/探索序列）；本节给"怎么真正整合"的操作心法。

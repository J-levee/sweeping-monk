# 混合方法卡（mixed-methods）

> 招式定位：建构/理论 / 解释机制 / 设计创制。学科轴：社科/教育/公卫/护理/医学。
> 与 `cards/design-science.md`（DSRM）、`sources/B6-design-based-research.md`（DBR）互补：混合方法是它们的常用"组合术"。

## 适用场景
- 既要"多大"（定量）又要"为什么/怎么回事"（定性）——单一范式答不全。
- 干预/项目评估：量看效果，质看机制与受者体验。
- 量表开发、实施科学、教育/组织变革——天然混合。

## 核心类型（Creswell & Plano Clark typology）
- **并行收敛（convergent / QUAN+QUAL）**：量质同时做，结果 triangulate 相互印证。
- **解释性序列（explanatory sequential：QUAN → qual）**：先做量（看效果），再用质解释"为什么"。
- **探索性序列（exploratory sequential：qual → QUAN）**：先做质（探构念/量表），再做量（验证推广）。
- **多阶段 / 嵌入式**：一个为主（加权），一个为辅补充。

## 操作步骤
1. **定主导 + 加权**：写清"以哪为主"（QUAN/QUAL 大写表示重）、为何混合。
2. **定接口点**：量→质 在哪交接？质如何取样（按量分组/极端案例）？
3. **并行或序列**排期；序列要留衔接窗口。
4. **整合（integration）**：不是"两份报告拼一起"——要在讨论层把量质结论合流（joint display / 矩阵）。
5. **效度威胁**：分别控量（信效度）与质（可信度），再加"混合效度（legitimation）"——设计/内部/外部/解释层的混合专属威胁。

## 前提
- 团队兼具量质能力，或协作；否则易"质的部分凑数"。
- 时间/资源够——混合比单一更贵。

## 常见坑
- **混合变"拼盘"**：量质各写各的，无整合层——等于没混合。
- **加权不清**：大写小写没标，读者不知以谁为主。
- **时序错配**：序列式里质性取样没按量结果走，接不上。
- **legitimation 忽略**：混合特有的"接口/解释"偏差未查。

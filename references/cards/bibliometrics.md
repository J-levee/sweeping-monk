# 文献计量卡（bibliometrics）

> 招式定位：描述 / 比较 / 关联（宏观结构层）。学科轴：通用（科学计量）。
> 贴合 SmartLib：12 亿文献库天然适合引文网络/突现词分析——老衲据此可快速定位领域结构与前沿。
> 根因提醒：老衲先问"你刻画的是领域演化、合作还是影响力"——问题不清，指标乱飞。

## 适用场景
- 用引文/文献数据刻画领域结构、演化趋势、合作网络、研究热点、影响力。

## 操作步骤
1. **定问题**：演化（时间）/ 合作（网络）/ 热点（突现）/ 影响力（指标）。
2. **取数**：多库引文（WoS/Scopus/CNKI/公司库），覆盖全免偏。
3. **建网络**：共被引 / 合作 / 文献耦合 / 共词。
4. **聚类与突现**：聚类定主题，突现词（Burst）定前沿。
5. **指标**：h 指数、中心性、被引百分位、期刊影响因子（慎用）。
6. **可视化**：图谱辅助，但需结合解读。
7. **质性补刀**：结合 `qualitative` / 批判阅读，网络发现需文献佐证。

## 前提假设
- 有引文数据库；问题属宏观结构层，非单篇深读。

## 常见坑（破绽速查）
- **引用偏见**：自引、语言（英文偏向）、学科差异扭曲指标。
- **引用量当质量**：高被引≠高质，可能争议大。
- **忽视非引文影响**：政策/产业影响不进引文。
- **数据覆盖不全**：单库结论偏，尤其中文文献。
- **过度解读网络**：聚类边界人为，硬下结论。

## 何时换招
- 要方法深度/单篇批判 → `qualitative` 或 research-playbook 第九节。
- 要检索具体文献 → 交还 literature-search / global-biblio-base。

## 经典出处（此招出自何典）
- 引文分析权威：Moed《Citation Analysis in Research Evaluation》（书单 #69）。
- 引文指标系列：Bornmann & Marx（书单 #70，OA，h 指数/分位数）。
- 脉络：引文分析由 Garfield（SCI）奠基、Moed 体系化；h 指数由 Hirsch(2005) 提出，但 Bornmann 警示其局限——老衲见"高被引=高质"必点引用偏见。
